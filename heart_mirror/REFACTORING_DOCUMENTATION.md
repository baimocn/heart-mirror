# 数据库服务层重构文档

## 重构概述

本次重构将数据库操作从 `models.py` 中分离出来，创建了一个独立的数据库服务层 `services/database_service.py`，实现了更好的代码组织和模块化。

## 主要变更

### 1. 创建数据库服务层

- **文件**: `services/database_service.py`
- **描述**: 实现了 `DatabaseService` 类，封装了所有数据库操作
- **功能**: 
  - 数据库连接管理
  - 表结构初始化
  - 数据CRUD操作
  - 统计数据计算

### 2. 重构 models.py

- **文件**: `models.py`
- **描述**: 将原来的独立函数重构为调用数据库服务层的方法
- **变更**: 
  - 移除了直接的数据库连接代码
  - 导入并使用 `db_service` 实例
  - 保持了原有函数签名，确保向后兼容

### 3. 更新路由模块

- **文件**: `routes/*.py`
- **描述**: 更新了所有路由模块，导入 `app` 对象和必要的装饰器
- **变更**: 
  - 从 `app` 模块导入 `app` 对象
  - 从 `app` 模块导入 `admin_required` 装饰器
  - 从 `app` 模块导入 `ADMIN_PASSWORD_HASH` 和 `verify_password_hash` 函数

### 4. 更新依赖

- **文件**: `requirements.txt`
- **描述**: 添加了缺失的依赖项
- **变更**: 
  - 添加了 `bcrypt==4.0.1`
  - 添加了 `python-dotenv==1.0.0`

## 技术架构

### 新架构

```
├── app.py              # 主应用文件
├── models.py           # 数据库操作接口（委托给服务层）
├── services/
│   └── database_service.py  # 数据库服务层
└── routes/
    ├── __init__.py
    ├── auth.py
    ├── quiz.py
    ├── donation.py
    ├── community.py
    ├── knowledge.py
    └── admin.py
```

### 数据流

1. 路由模块调用 `models.py` 中的函数
2. `models.py` 中的函数调用 `database_service.py` 中的对应方法
3. `database_service.py` 执行实际的数据库操作
4. 结果返回给路由模块

## 优势

1. **模块化**: 数据库操作集中管理，便于维护和扩展
2. **可测试性**: 服务层可以单独测试，不依赖于路由
3. **代码复用**: 避免了重复的数据库连接代码
4. **一致性**: 所有数据库操作遵循相同的模式
5. **可扩展性**: 易于添加新的数据库操作或更换数据库引擎

## 注意事项

1. **表结构变更**: 数据库服务层使用了更新的表结构，可能与旧版本不兼容
2. **字段映射**: 部分字段名称在服务层和模型层之间进行了映射，确保向后兼容
3. **性能考虑**: 服务层使用了相同的数据库连接模式，性能影响最小
