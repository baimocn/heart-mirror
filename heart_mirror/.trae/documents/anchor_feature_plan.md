# 心镜 · 锚点功能实现计划

## 1. 项目分析

### 当前代码结构
- **前端**：
  - `templates/index.html` - 结果页面HTML结构
  - `static/script.js` - 前端交互逻辑

- **后端**：
  - `services/database_service.py` - 数据库操作
  - `models.py` - 数据模型
  - `app.py` - API接口

- **数据库**：
  - `test_history`表 - 存储测试历史数据，包含device_id、fingerprint等字段
  - 目前没有专门的用户表来存储账号信息

### 现有功能
- **查看选择轨迹**：显示用户的测试选择历史
- **查看历史演化**：显示用户的测试结果演化趋势

## 2. 实现计划

### 2.1 数据库结构调整
1. **创建用户表**：
   - 表名：`users`
   - 字段：
     - `id` INTEGER PRIMARY KEY AUTOINCREMENT
     - `account_number` TEXT UNIQUE - 账号数字
     - `device_id` TEXT - 设备ID
     - `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP

2. **修改test_history表**：
   - 添加 `user_id` 字段，关联到users表的id

### 2.2 后端API实现
1. **添加用户管理API**：
   - `POST /api/anchor` - 处理锚点账号验证和创建
   - 功能：
     - 接收账号数字和device_id
     - 查询账号是否存在
     - 不存在则创建新用户
     - 存在则更新device_id关联
     - 返回用户信息

2. **修改现有API**：
   - 修改 `get_test_history` 函数，支持通过user_id查询
   - 确保轨迹和演化功能能够使用user_id获取数据

### 2.3 前端界面实现
1. **添加锚点功能区域**：
   - 在结果页面添加锚点输入框和确认按钮
   - 位置：核心功能按钮下方

2. **实现用户交互**：
   - 输入数字账号
   - 点击确认按钮触发验证/创建流程
   - 显示操作结果反馈

### 2.4 与现有功能集成
1. **与查看选择轨迹集成**：
   - 修改轨迹获取逻辑，优先使用锚点关联的用户数据
   - 确保通过锚点关联的测试数据能够正确显示

2. **与查看历史演化集成**：
   - 修改历史数据获取逻辑，使用锚点关联的用户数据
   - 确保新创建或同步的测试数据能够参与历史演化分析
